## 案例：一颗33Ω电阻救了一块板

一块板子上有一条100MHz时钟，从FPGA出发，走线8cm到一颗ADC。功能测试一路全过，但一上EMC测试就翻车——100MHz的谐波在300MHz、500MHz、700MHz一根根往上顶，全部超标。

拿示波器抓这条时钟线，波形问题一目了然：上升沿冲过头，过冲直顶到5V，而这颗芯片的IO供电才3.3V，绝对最大额定也只有3.6V。后面还拖着七八下越来越小的振铃，活像拿小锤敲了一下音叉，嗡嗡停不下来。

换晶振、查电源、改布线，忙活半天全没用。最后在FPGA那颗引脚上**串了一颗33Ω的电阻**，波形刷的一下就平了——过冲基本没了，振铃也消停了。EMC复测一次通过。

一颗几分钱的电阻，凭什么有这么大能耐？因为它治的不是晶振、不是电源，而是**这根走线的反射**。

## 反射从哪来

高速下，铜线不再是普通导线，而是一条有**特征阻抗**的传输线。板上单端常见50Ω，差分100Ω。只要这根线从头到尾阻抗都是50Ω，信号一路走得顺顺当当，什么事都没有。

麻烦出在**阻抗变了的地方**：走线突然变窄、打了个过孔、拐了个直角，或者到了末端那颗输入阻抗很高的接收芯片——只要前后阻抗对不上，一部分能量就没法继续往前，只能原路弹回去。这就是**反射**。

### 反射系数

反弹多少由反射系数决定，公式很简洁：

$$\Gamma = \frac{Z_L - Z_0}{Z_L + Z_0}$$

其中 $Z_L$ 是负载阻抗，$Z_0$ 是传输线特征阻抗。几种典型情况一算就明白：

| 末端情况                | 反射系数 $\Gamma$ | 效果                                                         |
| ----------------------- | ----------------- | ------------------------------------------------------------ |
| 完全匹配（$Z_L = Z_0$） | 0                 | 一点不反射，理想情况                                         |
| 开路（$Z_L = \infty$）  | +1                | 全部反弹，同向叠加到入射波上，电压直接翻倍——过冲就是这么来的 |
| 短路（$Z_L = 0$）       | -1                | 全部反弹，反向叠加                                           |
| 50Ω撞上75Ω              | 0.2               | 约两成能量弹回去                                             |

### 过冲与振铃

- **过冲**是第一次反射的峰值：信号跑到末端，遇到阻抗突变，一部分能量反弹回来，同向叠加到原始信号上，电压就冲过头了
- **振铃**是反射来回弹的余波：信号在源端和末端之间反复反射，一来一回，一次比一次弱，叠在波形上就成了那串逐渐衰减的振铃

## 什么情况下需要端接

不是每根线都得端接。关键看这根线**电气上长不长**——不是物理厘米数，而是信号在线上跑一趟的时间，跟它上升沿的快慢比起来长不长。

**判据：** 当信号单程跑一趟的时间超过其上升时间的1/6左右，就得把它当传输线考虑端接。

拿开头那条线算算：

- FR4板材的微带线，传播延迟大约每毫米6.7皮秒
- 8cm = 80mm，单程延迟 = 80 × 6.7 ≈ 536ps
- FPGA的上升时间约1ns，其1/6 ≈ 167ps
- 536ps >> 167ps，妥妥的电长线，**必须端接**

顺手推一个**临界长度**：让单程延迟正好等于上升时间的1/6，解出来大约：

$$L_{临界} \approx \frac{t_r}{6 \times 6.7\text{ps/mm}}$$

1ns的边沿算出来，临界长度约2.5cm。也就是说，这种边沿下超过2.5cm的走线就该端接了。**边沿越快，临界长度越短**——这也是为什么越是高速信号，越是短短一截也躲不过端接。

## 四种端接方式

思路只有两条：要么让阻抗连续（末端补一个跟传输线一样的阻抗），要么在源端把弹回来的能量吃掉。落到电路上就是四招。

### 第一招：源端串接（Series Termination）

在驱动器输出脚串一颗电阻，让这颗电阻加上驱动器的内阻，正好等于传输线特征阻抗。

开头那颗33Ω就是它——芯片内阻大约17Ω，加33Ω刚好凑到50Ω。

- **优点：** 几乎不耗静态电流，成本最低
- **最适合：** 点对点、单一负载的场景

### 第二招：末端并接（Parallel Termination）

在接收端对地接一颗电阻，阻值等于传输线特征阻抗，让末端阻抗直接匹配。

- **优点：** 匹配干净
- **适合：** 总线、多负载
- **缺点：** 这颗电阻常年挂着，静态功耗大

### 第三招：戴维南端接（Thevenin Termination）

末端用一上一下两颗电阻——一颗拉到电源，一颗拉到地。两颗并联值等于传输线特征阻抗，同时把电平偏置到中间。

- **适合：** 需要设偏置的场合，DDR的地址/控制线最常用

### 第四招：AC端接

末端一颗电阻串一颗小电容再接地。电容隔直流，只放交流——静态时不导通、不耗电，只有信号跳变的高频分量才被吸收。

- **适合：** 有直流电平又想省电的场合

### 差分线怎么端接

最常见的就是在差分对的末端跨接一颗100Ω的电阻，正好等于差分阻抗，把反射吸掉。讲究一点的用两颗50Ω分裂端接，中间再接一颗电容隔直。USB、HDMI、LVDS、以太网这些差分链路，末端跨接100Ω，板子上一定见过。

## 串阻阻值怎么定

源端串阻到底串多大，不是拍脑袋定的，得**扫着调**：

1. 从0Ω往上加
2. 用示波器盯着接收端波形

观察两种极端：

- **欠阻尼：** 阻值太小，过冲振铃压不干净
- **过阻尼：** 阻值太大，边沿被拖得软趴趴，上升时间变长

**临界阻尼**就是刚好把过冲压没、边沿又没明显变软的那个值——也就是你要的最优值。记住：临界阻尼是刚好，不是越大越稳。加过头，过冲是没了，但上升时间被拖长，时序余量反而被吃掉。

## 怎么验证端接效果

反射这东西全写在波形上。

### 第一件武器：示波器

探头尽量靠近接收端，因为反射在接收端看得最清楚。抓一个上升沿，量三样：

- **过冲百分比：** 一般要压到10%~20%以内
- **振铃：** 看几下能衰减掉，有没有跌破接收门限造成误触发
- **建立时间：** 看波形多久才稳定，太长会吃掉时序余量

端接前后各抓一张，对比效果一眼就见分晓。

**大坑：** 测的时候探头本身别引入新的反射。用长长的地线夹或者容性很大的探头，量出来的过冲振铃可能有一半是探头自己造的。高速场景尽量用**短地弹簧 + 低容探头**。

### 第二件武器：TDR（时域反射计）

TDR专门用来定位阻抗不连续到底在哪。它往线里发一个很陡的阶跃脉冲，然后看反射什么时候回来、幅度多大：

- 阻抗变高的地方（过孔、开路）：反射是正的，波形往上抬一个台阶
- 阻抗变低的地方（短路、容性负载）：反射往下掉

再根据反射回来的时间就能反推不连续点离探头多远——一条线上哪个过孔、哪个接插件在捣乱，TDR一扫就现形。

## 选型总结

先判这根线是不是电长线，不是就不用端接。是电长线，再分四种情况：

| 场景           | 端接方式     | 特点                   |
| -------------- | ------------ | ---------------------- |
| 差分线         | 末端跨接100Ω | 标准做法               |
| 点对点、单负载 | 源端串接     | 最省电、最省事         |
| 总线、多负载   | 末端并接     | 匹配干净，但有静态功耗 |
| 需要偏置       | 戴维南端接   | DDR地址/控制线常用     |
| 在意静态功耗   | AC端接       | 隔直流，只吸高频       |

## 常踩的坑

1. **端接电阻位置放错：** 源端串阻要紧贴驱动器出脚，末端并接要紧贴接收器。放远了等于白做
2. **阻抗没对准：** 要对准这条线真实的阻抗。阻抗控制没做好，线本身就不是50Ω，端接再准也没用
3. **探头引入假信号：** 高速测量时探头的地线和容性会引入额外的过冲和振铃，造成误判

## 总结

反射听着玄，其实就一句话：信号撞上阻抗不连续就往回弹。弹回来同向叠加就是过冲，来回弹的余波就是振铃。而EMC那些莫名其妙的谐波超标，很多时候就是这串振铃在辐射。

治它的钥匙也就两把：让阻抗连续，或者把反射能量吃掉。落到手上，就是源端一颗串阻、末端一颗并阻、或者一对戴维南分压——阻值位置对，几分钱的电阻就能把七八下的振铃摁成一条干净的边沿。

而这一切的前提是：你得先会用示波器和TDR把反射看出来。
