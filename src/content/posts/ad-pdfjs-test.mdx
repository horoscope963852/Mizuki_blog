这是一篇用于测试 PDF.js 库方案的博客文章。与 iframe 方案不同，PDF.js 提供了完全自定义的 PDF 阅读器界面。

## PDF.js 方案特点

PDF.js 是 Mozilla 开发的一个通用的 PDF 渲染库，具有以下优势：

1. **跨浏览器一致性**：所有浏览器显示效果完全相同
2. **高度可定制**：可以自定义工具栏、主题、布局等
3. **高级功能**：支持文本搜索、高亮、注释等
4. **暗色模式**：完美支持深色主题

上面的 PDF 预览器具有以下功能：

- ✅ **暗色模式**：自动适应系统主题
- ✅ **响应式设计**：适配移动端

## 与 iframe 方案的对比

| 特性           | iframe 方案 | PDF.js 方案 |
| -------------- | ----------- | ----------- |
| 实现难度       | ⭐ 简单     | ⭐⭐⭐ 复杂 |
| 跨浏览器一致性 | ⚠️ 一般     | ✅ 完美     |
| 自定义程度     | ❌ 低       | ✅ 高       |
| 暗色模式支持   | ❌ 无       | ✅ 支持     |
| 依赖大小       | 0           | ~2MB        |
| 移动端体验     | ⚠️ 一般     | ✅ 好       |

## 技术实现

PDF.js 方案的核心实现：

1. **安装依赖**：`pnpm add pdfjs-dist`
2. **创建 Svelte 组件**：封装 PDF.js 的渲染逻辑
3. **配置 Worker**：设置 PDF.js 的 worker 文件路径
4. **响应式设计**：支持暗色模式和移动端
