仅限自用。

## 第一步：修改 Xray 配置

**编辑 Xray 配置文件**：

```bash
sudo nano /usr/local/etc/xray/config.json
```

**修改内容**： 找到 `inbounds` 中的 `password` 和 `path` 字段，将其替换为你的新凭据。

```json
// 找到这里并修改
"clients": [
  {
   "password": "New_Pwd"
  }
]
// ...
"wsSettings": {
  "path": "/_new_a1b2c3"
}
```

**重启 Xray 服务**：

```bash
sudo systemctl restart xray
```

## 第二步：修改 Nginx 配置

因为 Nginx 负责反向代理 WS 路径，Xray 的路径改了，Nginx 也必须同步修改，否则流量进不来。

```bash
sudo nano /etc/nginx/conf.d/xtkx.site.conf
```

```nginx
# 1. 暂时关闭旧节点的路径
location /_campus {
    # 直接返回 403 拒绝访问。以后想恢复时，把这行删掉并恢复下面的反代代码即可
    return 403; 

    # if ($http_upgrade != "websocket") { ... }
    # proxy_pass http://127.0.0.1:1122;
    # ...
}
```

**重载 Nginx 服务**：

```bash
sudo nginx -t
sudo nginx -s reload
```
